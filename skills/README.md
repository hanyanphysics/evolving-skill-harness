# Skills

This directory contains global reusable skill placeholders.

Most skill files are intentionally empty for now.

The user will later decide which skills to write first.

## Categories

```text
physics/
```

Physics and scientific research skills.

```text
coding/
```

General coding and software engineering skills.

```text
meta/
```

Skills about building, revising, auditing, and maintaining skills.

## Rule

Do not add substantive skill content unless the user explicitly asks.

## Revision notice

Whenever reusable skill behavior in this directory is added, edited, revised, deleted, renamed, validated, or promoted, Codex must report it in the project conversation with a visible ASCII box:

```text
+------------------------------------------------------------+
| New skill learned!                                         |
| At skills/xxx/xxx.md, the following skill has been added:  |
| [Description of the skill]                                 |
+------------------------------------------------------------+
```

Use the real path and a concise description of the added or revised skill. If the skill was changed rather than newly created, write "added or revised" in the path line.

Before the final response, check whether any file under `skills/**` changed reusable skill behavior, metadata, validation, or instructions. If yes, include the box.
