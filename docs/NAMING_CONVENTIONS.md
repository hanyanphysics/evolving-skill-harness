# Naming Conventions

Use clear lowercase filenames with underscores.

Preferred:

```text
scientific_rigor.md
theorem_writing.md
lesson_capture.md
project_memory.md
```

Avoid:

```text
ScientificRigor.md
theorem-writing-final-v2.md
misc.md
notes.md
```

## Skill names

Skill names should usually describe an action or workflow.

Preferred:

```text
theorem_writing.md
literature_search.md
failure_analysis.md
```

Less preferred:

```text
theorems.md
papers.md
mistakes.md
```

## Project-specific files

Project-local files should clearly indicate that they are project-specific.

Examples:

```text
project_memory.md
local_context.md
project_specific_notation.md
```

## Archive policy

Do not delete old skills casually.

When an archive system is later added, obsolete skills should be moved to an archive rather than silently removed.
